Система внутреннего документооборота

OAuth сервис для аутентификации пользователей

О системе

Это имитация системы внутреннего документооборота компании, которая поддерживает OAuth аутентификацию для внешних приложений.

Данная система позволяет пользователям безопасно авторизоваться и предоставить доступ внешним приложениям к своим документам.

OAuth 2.0 Endpoints

GET /oauth/authorize

Точка входа для авторизации пользователя. Перенаправляет на страницу входа.

Параметры: response_type=code, client_id, redirect_uri, scope, state (опционально)

POST /oauth/login

Обработка формы входа пользователя.

POST /oauth/token

Обмен кода авторизации на токены доступа.

Поддерживает: grant_type=authorization_code и grant_type=refresh_token

API-эндпоинты документов

GET /api/documents

Получение списка документов текущего пользователя.

GET /api/documents/{document_id}

Получение конкретного документа по ID.

POST /api/documents

Создание нового документа.

POST /api/token/validate

Валидация токена доступа. Используется внешними сервисами для проверки токена.

GET /api/user/me

Получение информации о текущем пользователе.
